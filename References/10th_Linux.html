Linux

Why?

Advantages

  • Simplicity: simple to install, simple to use -- it just works
  • Comfort: automatic centralized updates of ALL applications
  • Security: no viruses, no malware, no bloatware, no licensing worries, no waiting for important security updates
  • Price: it's not only free in the meaning of liberty, it's also free in the meaning of free beer
  • Choice: thousands of free and open source applications are available for almost every purpose, and can be automatically installed through an integrated package browser.
  • Community: people help and support each other, trying new things, exploring possibilities, learning technologies, and so on.
  • It's not THAT different from "mainstream commercial systems", but basically the same desktop environment as usual.
  • If you are used to one of these mainstream commercial systems, don't worry, there are many good alternatives for your favorite mainstream proprietary applications.

Handicaps

  • Sometimes lacking support from commercial service providers.
  • Sometimes difficulties with very old or very new or very exotic hardware, namely:
    graphics cards, monitors, printers, tv cards.
  • For some excessive mainstream gaming, you might want to keep a dual-boot system.
  • If you are very used to a certain other system, and you don't like to learn something new, then better stay with it because there is a difference.

Fault Recovery

  • What to do if some application doesn't promptly respond? First, you should keep calm. Try to normally shut it down. After a minute or so (depending on the system load) you might let the desktop environment kill the application via window decoration's close button. You are likely to be asked if you really want to kill it, or to wait some more.
  • Anyway, you can launch your favorite task manager like top on terminal or gnome-system-monitor that allows you to watch the resources and kill or terminate applications.
  • If the graphical environment doesn't respond, then my next attempt would be to switch to a virtual console via CTRL-ALT-F1 or -F2, log in with superuser privileges, and to run top there.
  • If you aren't familiar with the console or top or simply aren't in the mood to log in, you can simply press CTRL-ALT-DEL in a virtual console to initialize a soft reboot.
  • A magic SysRq key combination (Alt + SysRq + some key) can send various commands to the kernel even if the system seems to be frozen. A common idiom to perform a safe reboot of a Linux computer which has otherwise locked up, the QWERTY mnemonic "Raising Elephants Is So Utterly Boring", "Reboot Even If System Utterly Broken" or simply remembering the word "BUSIER" backwards, is often useful: unRaw, tErminate, kIll, Sync, Unmount, reBoot. Wait a few seconds each.

Data Recovery

Recovery from Physically Damaged Drive

  1. apt-get install gddrescue - gddrescue with logfiles allows to be called multiple times and rescues only what is left untried; don't mistake for dd_rescue
  2. ddrescue -s50M /dev/hdc hdimage.bin logfile.log - rescue most important parts first: copy first 50 MB
  3. ddrescue -s1M -r3 -d /dev/hdc hdimage.bin logfile.log - rescue important, try hard: copy first 1 MB, max-retries 3 times, direct access
  4. ddrescue -i39G -s1G /dev/hdc hdimage.bin logfile.log - rescue end of disk (example disk would have a size of 40 GB)
  5. ddrescue -i19G -s2G /dev/hdc hdimage.bin logfile.log - rescue middle of disk (example disk would have a size of 40 GB)
  6. ddrescue -n /dev/hdc hdimage.bin logfile.log - rescue whole disk quickly: no-split (stop after trimming, saves much time)
  7. ddrescue -dr3 /dev/hdc hdimage.bin logfile.log - rescue whole disk thoroughly: direct access, max-retries 3
  8. ddrescue -dRT /dev/hdc hdimage.bin logfile.log - re-try everything exhaustively: direct access, re-trim, try-again
  9. then you can make a backup copy of the image and mount it
  10. see also: magicrescue to scan for lost files

Move /home to another mountpoint

  1. check size of /home: du -sh /home
  2. create an empty formatted partition that is large enough
  3. log off, use terminal
  4. find new partition: fdisk -l
  5. mount new partition: mkdir /mnt/tmp; mount /dev/sda6 /mnt/tmp
  6. sync content: rsync -avx --progress /home/ /mnt/tmp (cp is faster but don't guarantees consistency)
  7. mount new /home: mount /dev/sda6 /home
  8. check result: du -sh /home; mount|grep /home
  9. unmount tmp: umount /home
  10. remove old data: rm -rf /home/* (don't remove the folder itself)
  11. get UUID of new partition: blkid|grep sda6
  12. enter line to /etc/fstab: UUID=f72aac23-092f-4484-b35c-f222c106d716 /home ext3 defaults 0 2
  13. leave chroot: exit
  14. reboot

File System Hierarchy

Info

  • Wikipedia - Filesystem_Hierarchy_Standard
  • Ubuntuusers.de Wiki - Verzeichnisstruktur (de)

/bin

  • This folder contains the most basic applications that have to be present on every Linux system, like the shell, tools for file and process management, and for string manipulation.

/boot

  • /boot/grub/menu.lst - grub 1.x boot menu

/dev

  • /dev/fd - floppy drive
  • /dev/hda - first hard disk drive
  • /dev/null - data sink
  • /dev/sdb3 - third partition on second scsi (or sata) disk drive
  • /dev/shm - virtual ramdisk (tmpfs), default size is half of RAM
  • /dev/urandom - unblocking random, faster than /dev/random, but much slower than /dev/zero
  • /dev/zero - data source (binary zero)

/etc

  • /etc/apt/preferences - priorities of package, allows pinning of versions
  • /etc/apt/sources.list - package sources; examples:
    deb http://ftp.de.debian.org/debian woody main contrib non-free
    deb file:///tmp/download ... ?
    deb cdrom:[Ubuntu 9.04 _Jaunty Jackalope_ - Release i386 (20090420.1)]/ jaunty main restricted
  • /etc/conf.d/net - network configuration
  • /etc/ddclient.conf - client for dynamic dns; example:
    protocol=dyndns2, use=if, if=ppp0, server=members.dyndns.org, login=foo, password='bar', myaddress1.dyndns.org,myaddress2.dyndns.org
  • /etc/default/... - default configurations (?)
  • /etc/environment - $PATH
  • /etc/exports - nfs shared directories (?), e.g. "/home *.domain.de(rw)" - user id counts for access rights
  • /etc/fstab - file system table
  • /etc/gdm/PostSession/Default - run on gnome session end (user logout); e.g. 'truecrypt -d'
  • /etc/hostname - name of the computer
  • /etc/hosts - hostname-to-address mappings
  • /etc/iftab - assigns persistent names to network interfaces. Example: eth0 mac 00:A0:24:51:8F:F4. (Ubuntu only till 7.04?)
  • /etc/inittab - defines what each runlevel does (?)
  • /etc/init.d/ - init scripts for running services. Examples:
    sudo /etc/init.d/sshd [start|stop|restart]
    sudo /etc/init.d/hal restart
  • /etc/issue - is displayed on console login prompt (before a user logs on), can hold some escape sequences like \n=nodename; issues.net is used by SSH
  • /etc/mke2fs.conf - default settings for new file systems
  • /etc/modules - kernel modules loaded at boot, e.g. fuse, it87
  • /etc/motd - is displayed on console login (after a user logs on), can hold some escape sequences like \n=nodename; is re-created on boot
  • /etc/motd.tail - is appended to motd (optional)
  • /etc/mtab - table of mounted file systems
  • /etc/network/interfaces - network interfaces
  • /etc/openoffice/sofficerc - set Logo=0 for speedup
  • /etc/rc?.d/ - links used by init and managed by update-rc.d
  • /etc/rc.local - is executed after each runlevel change (?)
  • /etc/resolv.conf - address resolution, DNS server etc.
  • /etc/ssh/sshd_config - sshd config; e.g. set 'PermitRootLogin no'
  • /etc/skel/ - template for new user profiles
  • /etc/sudoers - list of who is allowed to run what as super user
    %admin ALL=NOPASSWD: /bin/mount # allow members of group admin to not need a password (note that later entries override this)
    %admin ALL=NOPASSWD: /bin/umount
    %admin ALL=NOPASSWD: /usr/bin/truecrypt
  • /etc/sysfs.conf - tune hardware settings, e.g. CPU frequency scaling:
    devices/system/cpu/cpu0/cpufreq/scaling_governor = ondemand
    devices/system/cpu/cpu0/cpufreq/ondemand/up_threshold = 90 # default 95; 90 to better trigger flash videos in firefox (run with 'taskset -c 0')
  • /etc/udev/rules.d/70-persistent-net.rules - e.g. assign ETH0 to MAC address
  • /etc/X11/xorg.conf - X configuration: monitors, graphics cards, keyboards, mice
  • /etc/xdg/autostart - autostart .desktop files for all users.

Xubuntu

  • Setup "Start-Menu" key (Super_L) to open Xfce-Menu:
    1. create keyboard shortcut in xfce settings (will be "Super+Super_L")
    2. edit ~/.config/xfce4/shortcuts/name_of_your_shortcuts.xml
    3. remove "Super+" from the respective entry
    4. open and close shortcut configuration, or restart X to activate
    Problem: the key doesn't work as modifiert (as in Super+e) any more.
    Workaround: don't use Super_L for menu, but something like Super+w.
  • Enable Gigolo to open Thunar with doubleclick on mounted SMB volume:
    1. # apt-get install gvfs-fuse fuse-utils - install GVFS backend so that GVfs mounts the remote resources in ~/.gvfs
    2. # gpasswd -a username fuse - add user to the fuse group
    3. $ nano ~/.local/share/applications/defaults.list
      x-directory/gnome-default-handler=Thunar.desktop
      inode/directory=Thunar.desktop
      x-directory/normal=Thunar.desktop       - enable Thunar to open folders by gvfs-open
    4. You might have to relaunch your session.
    5. See Open resources in Thunar on Xfce 4.4 and 4.6 (uvena.de)

Knoppix

  • http://knopper.net/knoppix/
  • most important boot options:
    failsafe - minimum hardware detection; if this does not run, the is almost no hope
    knoppix 2 - runlevel 2, textmode only
    knoppix lang=de - specify language/keyboard
    knoppix screen=1280x1024 - use specified screen resolution for X
    knoppix xvrefresh=60 - use 60 Hz vertical refresh rate for X
    knoppix fb1024x768 - use fixed framebuffer graphics
    knoppix noapic - disable apic
    knoppix nousb nofirewire - disable usb and firewire
    knoppix vga=normal - disable framebuffer mode
    knoppix nodma - disable dma (very slow)
    knoppix noideraid - disable raid detection (if raid available)
    knoppix nopcmcia - disable pcmcia detection (on laptops)
    knoppix acpi=off - disable acpi
    knoppix pnpbios=off - disable pnp bios initialization
